## Формулировка задачи

### Описание

Руководитель отдела сопровождения клиентов хочет оценить, насколько хорошо аккаунт‑менеджеры справляются с задачей пролонгации договоров с клиентами.  
Нужно подготовить отчёт о пролонгациях за 2023 год по каждому менеджеру и по отделу в целом. 
### Определения коэффициентов пролонгации

Используются два коэффициента пролонгации (на уровне отдела и на уровне каждого менеджера). 
1. **Коэффициент пролонгации в первый месяц (K1)**  
   Отношение суммы отгрузки проектов, пролонгированных в первый месяц после завершения, к сумме отгрузки последнего месяца реализации всех проектов, завершившихся в предыдущем месяце.

2. **Коэффициент пролонгации во второй месяц (K2)**  
   Отношение суммы отгрузки проектов, пролонгированных во второй месяц после завершения, к сумме отгрузки последнего месяца проектов, которые не были пролонгированы в первый месяц. 

Пример для оценки пролонгации в мае:

- K1 (пролонгация в первый месяц):  
  - База: проекты, завершившиеся в апреле (сумма отгрузки за апрель).  
  - Факт: сумма отгрузки тех же проектов в мае (при наличии отгрузки в мае).  
  - K1 = «сумма за май» / «сумма за апрель».

- K2 (пролонгация во второй месяц):  
  - База: проекты, завершившиеся в марте, у которых нет отгрузки в апреле (сумма отгрузки за март).  
  - Факт: сумма отгрузки этих проектов в мае (есть отгрузка в мае и нет в апреле).  
  - K2 = «сумма за май» / «сумма за март». 

### Исходные данные

Есть два набора данных. 

1. `prolongations.csv`  
   - `id` – идентификатор проекта.  
   - `month` – последний месяц реализации проекта.  
   - `AM` – ФИО ответственного аккаунт‑менеджера (эти данные считаются первичными относительно `Account` в `financial_data.csv`).

2. `financial_data.csv`  
   - `id` – идентификатор проекта.  
   - `Причина дубля` – причина, по которой строки с одинаковым `id` встречаются несколько раз.  
   - Колонки с названиями месяцев – сумма отгрузки проекта в соответствующий месяц.  
   - `Account` – ФИО ответственного аккаунт‑менеджера. 

Дополнительные значения в колонках месяцев в `financial_data.csv`: 

- `в ноль` – отгрузка проекта в данном месяце равна 0; для расчёта коэффициентов пролонгации нужно взять отгрузку предыдущего месяца (если все части оплаты в этом месяце равны 0).  
- `стоп` – проект закончился до истечения срока договора; если у проекта есть `стоп` в последний месяц реализации или ранее, такой проект исключается из расчёта пролонгаций.  
- `end` – аналог `стоп`, также означает досрочное окончание проекта и исключение из пролонгаций.

### Требуется сделать

1. Рассчитать коэффициенты пролонгации для каждого аккаунт‑менеджера и для отдела в целом:
   - по каждому месяцу 2023 года;  
   - за весь год (агрегированное значение по году). 

2. Сформировать аналитический отчёт в Google Sheets или Excel, на основании которого руководитель отдела сможет принимать управленческие решения:
   - обязательно включить K1 и K2 по каждому менеджеру и по отделу;  
   - можно добавить дополнительные метрики и визуализацию (диаграммы, сводные таблицы и т.д.).  

3. Соблюсти шаблон оформления итоговой таблицы (структура листа и основные поля), авторские дополнения можно вынести на отдельные листы.

4. Обеспечить читаемость и понятность отчёта для руководителя отдела (понятные названия метрик, читаемые подписи, группировка по менеджерам и месяцам).
