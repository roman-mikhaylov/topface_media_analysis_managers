# Customer Contract Prolongation Analysis

This project analyzes how efficiently account managers prolong customer contracts after their initial end date. It calculates first‑ and second‑month prolongation coefficients (K1/K2) per account manager and for the whole department using raw financial exports. The results feed an executive‑level Excel/Google Sheets report with dashboards showing monthly dynamics and performance by account manager. 

## Business problem

A head of Customer Success wants to evaluate how well account managers handle contract prolongations (renewals) for existing customers.  
The goal is to measure prolongation performance across 2023 and provide a management‑ready report with clear KPIs and visualizations. 

The company uses two prolongation coefficients:

- **K1 (1st‑month prolongation)** – share of revenue from projects prolonged in the first month after contract end.  
- **K2 (2nd‑month prolongation)** – share of revenue from projects prolonged in the second month after contract end, among projects that were not prolonged in the first month.

Coefficients are calculated:

- for the whole department;  
- for each individual account manager (AM). 

## Data sources

Two input CSV files are used. 

### `financial_raw.csv`

Monthly financial data per project.

- Row = project `id`.  
- Columns: `Account`, `Причина дубля` (duplicate reason), monthly revenue columns (e.g. November 2022 … February 2024).  
- Cell values: numeric amounts or special markers: `стоп`, `end`, `в ноль`.

These markers mean:

- `стоп` / `end` – the project was stopped early; such projects are fully excluded from prolongation analysis.  
- `в ноль` – zero revenue in this month; when all payment parts in the month are zero, the prolongation coefficient uses the previous month’s revenue instead. 

### `prolongations_raw.csv`

Contract metadata for prolongation analysis.

- Row = project `id`.  
- Columns: last month of contract realization `month` and responsible account manager `AM`.

All calculations are performed on normalized versions of these tables (`financial_long_clean.csv`, `prolongations_norm.csv`), which are generated by the script. 

## Prolongation logic (K1 / K2)

Full details and step‑by‑step logic are described in `code_and_logic.md`. Below is the core idea. 

- For each project we define the last realization month `last_month_code` based on `prolongations_raw.csv`.  
- Projects that contain `стоп` or `end` in any month are excluded from the analysis.  
- For a target month \(M\):
  - K1 uses projects with last month \(L = M - 1\).  
  - K2 uses projects with last month \(L = M - 2\), but only those that had no prolongation (no revenue) in month \(M - 1\). 
Department‑level coefficients:

- \(K1(M) = \text{prolongation revenue in month } M \,/\, \text{base revenue of projects with } L = M-1\).  
- \(K2(M) = \text{prolongation revenue in month } M \,/\, \text{base revenue of projects with } L = M-2 \text{ and no prolongation in } M-1\). 

The same formulas are applied at the account‑manager level, filtering data by AM.

topface_media_analysis_managers/

## How to run

### 1. Clone the repository

```bash
git clone https://github.com/roman-mikhaylov/topface_media_analysis_managers.git
cd topface_media_analysis_managers
```


### 2. Create and activate a virtual environment

macOS / Linux:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

Windows (PowerShell):

```powershell
python -m venv .venv
.\.venv\Scripts\activate
```


In your IDE (e.g. VS Code), select the Python interpreter from `.venv` before running the script. [support.microsoft](https://support.microsoft.com/en-us/office/save-your-workbook-92e4aae0-452d-497f-a470-570610ff720a)

### 3. Install dependencies

The project uses **pandas** for data manipulation. 

```bash
pip install --upgrade pip
pip install pandas
```


For working with Excel output you may also install:

```bash
pip install openpyxl
```


### 4. Prepare input files

Place the input CSV files into the `data/` folder:

- `financial_raw.csv` – raw financial data export;  
- `prolongations_raw.csv` – projects with last realization month and responsible AM. 

Expected folder structure:

```text
topface_media_analysis_managers/
  data/
    financial_raw.csv
    prolongations_raw.csv
```


### 5. Run the analysis

From the repository root, with the virtual environment activated:

```bash
python py/manager_analysis.py
```


The script will:

- normalize `financial_raw.csv` and `prolongations_raw.csv`;  
- exclude projects with `стоп` / `end` statuses;  
- compute bases and prolongation amounts (K1/K2) per month for the department and each AM;  
- write aggregated results into the `out/` folder. 

### 6. Output files

After successful execution, the following files are created in `out/`:

- `department_from_prol_2023.csv` – bases and K1/K2 coefficients per month for the whole department;  
- `am_from_prol_2023.csv` – the same metrics by account manager × month;  
- `am_from_prol_k1_pivot_2023.csv` – K1 matrix (rows: AMs, columns: months);  
- `am_from_prol_k2_pivot_2023.csv` – K2 matrix (rows: AMs, columns: months). [github]

These files are used as data sources for the final Excel / Google Sheets management report and dashboards. 

## Management report

The executive‑level report is built in Excel / Google Sheets based on the CSV outputs from `out/`:

- sheet “Department” – monthly K1 and K2 dynamics for the whole portfolio;  
- sheet “Account managers” – K1/K2 metrics per AM;  
- dashboards – charts comparing managers, tracking month‑over‑month changes and summarizing annual performance. 

The final Excel file (for example `report/contract_prolongation_report_2023.xlsx`) can be stored in the repository and linked from this README, e.g.:

```markdown
- `report/contract_prolongation_report_2023.xlsx` — management report with dashboards based on the analysis results.
```
